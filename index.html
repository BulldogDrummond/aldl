Aldl

A handler for gm obd-i aldl serial diagnostic systems.

Download as .zip Download as .tar.gz View on GitHub

This is a program designed to manage aldl datastreams from the 8192 baud series of OBD-I GM ECMs.

Right now it's fairly functional. Plugins and definition files need to be written, and more testing needs to be done.

The main goal is to turn small low-power linux devices such as the raspberry pi into a powerful auxiliary dashboard or perpetual datalogger:

  • Maintain stable diagnostic mode, with methods that are adaptable to all GM ECMs.
  • Low memory and CPU usage with pure optimized C
  • Portable to at least x86 and ARM
  • Parse multiple datastreams efficiently with maximum throughput allowable
  • Accurate unit conversions for all known data that are transparent to plugins
  • Stupid simple plug-in API that requires no knowledge of the datastream
  • Parallel processing of output plugins with a single serial input
  • Script to convert ADF format spec

Possible plugins and applications for target devices include:

  • ncurses or GTK display interfaces for supplimental dashboards
  • lightweight multi-format dataloggers
  • warning buzzer or warning light widgets
  • simple scripting engines, which, combined with a GPIO header, could allow this to become a fairly capable piggyback ECM for boost control, cutout actuation, or whatever
